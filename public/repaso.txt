ENTORNO: NODE (npm) --> ng cliente de angular
COMPONENTE: Apartado de página (CSS+HTML+JS(TS)), "etiqueta personalizada"
ENRUTAMIENTO: 

app.routes: fichero que asocia rutas con componentes
<router-outlet>: componente de referencia donde se anclan el resto
routerLink, routerLinkActive: directivas (atributos de etiquetas)
Módulo de rutas: RouterModule --> módulo: agrupación funcional de clases

Depedencias de nuestra app, van en el fichero package.json

SPA: Single Page Application . Sólo exite el index.html

Ciclo de vida del componente: constructor, ngOnInit...ngOnDestroy

@if @for (antaño el ngIf y el ngFor): introducir bucles y secciones condcionales en el html

COMUNICACIÓN TS <---> HTML Componente - Plantilla

{{interpolación}} 
[property binding] img, class, disabled
(event binding) listener-callback
[(ngModel)] two way binding

SERVICIO: Clase que sirve datos al Componente "Proveedor"
Para poder conectarnos por HTTP, necesitamos en el fichero app.config
habilitar el módulo httpClientModule provideHttpClient()

INYECCIÓN DE Depedencias

Por defecto cuando angular arranca busca las clases anotadas 
como servicios (Injectable) y las instancia. 

Posteriormente desde el componente que necesita ese servicio lo declaramos como un atributo en el constructor o bien utilizamos la función INJECT para acceder a la instancia 

Los servicios siguen el patron Singleton! (sólo hay una instancia)

NGICONS - librería para iconos vectoriales
Usando npm install https://github.com/ng-icons/ng-icons

REDUX - NG RX 

"nos da memoria entre pantallas/componentes" y además, trazabilidad de un proceso
1) instalamos librerías 
npm i @ngrx/store@1
npm i @ngrx/store-devtools@19

2) CREAMOS ESTADO, ACTIONS Y REDUCERS

2.1 
Modelamos Estado - Qué datos quiero ir guardando
ContratacionState (DatosPersona, DatosCoche, isContratado)
y Defino un estado inicial (persona '', datosCoche '', contratado 'false')

2.2
Defino los eventos (llamadas ACCIONES) que alteran el estado (inicial) de la app
"GuardarPersona" -- cuando el usuario meta sus datos en el paso 1 del formulario y le dé a guardar
"GuardarCoche" -- cuando el usuario meta su coche en el paso 2 del formulario y le dé a guardar
"MarcarContrado" -- cuando el usuario active contratción en el paso 3 del formulario y le dé a contratar

2.3
Defino las acciones (llamadas REDUCER) consecuencia de haberse disparado un evento (Acción)
Son funciones que sí que van a operar sobre el estado actual y van a crear uno nuevo
Por cada acción/evento, programamos el REDUCER 3
Generando un NUEVO estado en cada una
